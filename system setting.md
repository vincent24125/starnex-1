# 1. 引言

## 1.1 目的

本文档为 **AI赛车仪表系统（AI Racing Dashboard, AIS）** 的软件需求规格说明书（Software Requirements Specification, SRS）。本文档旨在：

1. **明确系统边界**：定义AI赛车仪表系统的功能范围、性能指标及接口规范；
2. **指导开发实现**：为硬件电路设计、嵌入式固件开发、移动端App开发及云端服务开发提供统一的需求基线；
3. **支撑测试验证**：作为系统验证测试（V&V）、出厂检验、赛道实测的验收依据；
4. **管控项目风险**：通过优先级划分（必备/扩展/可选）和降级策略设计，降低OBD参数兼容性差异带来的交付风险。

本文档的预期读者包括：系统架构师、嵌入式软件工程师、硬件工程师、测试工程师、产品经理、项目经理及售后技术支持人员。

## 1.2 范围

### 1.2.1 产品范围

AI赛车仪表系统（AIS）是一款面向汽车运动爱好者、赛道日参与者及专业赛车队的高性能车载数据终端。产品通过车辆OBD-II诊断接口接入整车CAN总线，结合外置u-blox高精度GNSS模块与六轴IMU传感器，实现以下核心能力：

- **实时监测**：25项动力总成及底盘关键参数的实时采集与可视化；
- **运动分析**：基于GPS+IMU融合的高精度车辆动力学数据采集（G值、轨迹、姿态）；
- **赛道辅助**：预加载赛道地图、自动计圈、最优圈速比对、AI语音弯道提醒；
- **数据上云**：通过4G蜂窝网络将行驶数据、故障码实时上传至云端服务器；
- **社交竞技**：云端圈速排行榜、个人历史数据分析、车友数据分享；
- **远程运维**：支持OTA固件升级、远程故障诊断、参数配置下发。

### 1.2.2 功能边界

**系统包含的功能**：
- OBD-II ISO 15765-4 CAN协议数据请求与解析；
- u-blox UBX协议GNSS数据解析（25Hz刷新率）；
- MPU6050 I2C接口IMU数据采集与姿态解算；
- TFT LCD多页面图形化仪表渲染（转速表、数字仪表、赛道地图）；
- LED可编程换挡提示灯与蜂鸣器提醒；
- 本地TF卡数据日志记录（CSV/二进制格式）；
- 蓝牙BLE双模通信（手机App实时数据透传）；
- 4G Cat.1/Cat.4模块MQTT数据上云与OTA；
- AI语音合成与识别（TTS/离线关键词唤醒）。

**系统不包含的功能**：
- 车辆ECU刷写或标定修改（仅读取，不写入动力相关参数）；
- 主动车辆控制（不与转向、制动、油门执行器交互）；
- 后装传感器的数据闭环控制（温度传感器仅采集，不参与控制）；
- 视频/图像采集与处理（无摄像头接口设计）。

## 2.3 用户特征

### 2.3.1 用户角色定义

| 用户角色 | 技术背景 | 使用场景 | 核心诉求 |
|---------|---------|---------|---------|
| **赛道日爱好者** | 中等，了解基本车辆参数 | 赛道日驾驶、圈速挑战 | 易读仪表、快速圈速反馈、换挡提醒 |
| **专业赛车手/车队** | 高，深度理解车辆动力学 | 比赛数据记录、调校分析 | 高精度数据、完整日志、云端分析 |
| **改装车玩家** | 中高，关注动力参数 | 街道/山路驾驶、性能监测 | 涡轮压力、空燃比、油温等改装关注参数 |
| **普通车主** | 低，基础驾驶知识 | 日常驾驶、故障检测 | 简洁界面、故障码读取、电池电压 |
| **系统管理员/运维** | 高，嵌入式开发背景 | 固件维护、批量升级、远程诊断 | OTA管理、日志分析、设备状态监控 |

## 2.5 设计与实现约束

### 2.5.1 硬件资源约束

**STM32F407VGT6资源限制**是系统设计的核心约束：

| 资源 | 总量 | 预分配 | 剩余 | 约束说明 |
|------|------|--------|------|---------|
| Flash | 1024KB | Bootloader 64KB + 固件 768KB + 配置区 64KB + OTA缓存 128KB | ~0KB（需精细规划） | OTA缓存区复用策略 |
| SRAM | 192KB | FreeRTOS/Heap 64KB + LCD帧缓冲 64KB + 任务栈/全局 48KB + 缓冲区 16KB | ~0KB | 帧缓冲需双缓冲或优化 |
| DMA通道 | 16路 | UARTx4 + SPIx2 + I2Cx1 + SDIOx1 + ADC | 紧张 | DMA调度冲突风险 |
| CAN接口 | 2路 (bxCAN) | CAN1接OBD，CAN2预留 | CAN2可用 | 双CAN同时运行需滤波器规划 |
| UART | 6路 | GPS/4G/蓝牙/调试/语音/预留 | 紧张 | 波特率配置需错开 |
| ADC通道 | 16路 | 温度传感器x4 + 电池电压x1 + 预留 | 可用 | 扫描模式+DMA |
| TIM定时器 | 14路 | 系统tick/换挡灯PWM/蜂鸣器/输入捕获 | 可用 | 高级定时器用于LCD背光 |

**关键约束推论**：
- **内存约束**：192KB SRAM无法同时支持800x480x16bit双缓冲（需307KB），必须采用**单缓冲+DMA2D传输**、**局部脏区域刷新**或**降低色深至8bit索引色**策略；
- **Flash约束**：OTA升级需128KB缓存区，考虑与配置区动态复用，或采用**A/B分区升级**方案；
- **DMA约束**：GNSS 25Hz高频数据（约1.5KB/s）必须走UART DMA环形缓冲区，避免CPU轮询开销；
- **CAN约束**：OBD请求采用ISO 15765-4 TP层（多帧传输），需精确控制发送时序与流控帧。

## 3.1 功能需求

### 3.1.1 OBD/CAN数据采集子系统 (SS-01)

#### 3.1.1.1 功能需求总览

本子系统负责通过车辆OBD-II接口接入CAN总线，依据ISO 15765-4协议与车辆ECU通信，完成25项动力总成参数的周期性采集、故障码诊断及参数可用性动态管理。

**[FR-OBD-001] OBD-II CAN总线连接与初始化**

| 属性 | 描述 |
|------|------|
| **需求ID** | FR-OBD-001 |
| **需求描述** | 系统上电后，自动完成CAN控制器初始化（bxCAN配置）、CAN收发器使能、OBD-II诊断会话建立（默认会话->扩展诊断会话）。支持CAN波特率自动探测（250kbps/500kbps）。 |
| **优先级** | P0 必备 |

**[FR-GPS-005] 姿态角解算（互补滤波/Mahony）**

| 属性 | 描述 |
|------|------|
| **需求ID** | FR-GPS-005 |

**[FR-GPS-006] 横向/纵向G值计算**

| 属性 | 描述 |
|------|------|
| **需求ID** | FR-GPS-006 |
| **需求描述** | 基于IMU加速度数据与姿态角，计算车辆坐标系下的横向（Lateral）与纵向（Longitudinal）G值。 |


```c
CK_A = 0; CK_B = 0;

    CK_B = CK_B + CK_A;
}
```

## 附录D：IMU数据融合算法说明

### D.1 坐标系定义

```
车身坐标系（右手坐标系）：
  X轴正向：车头前进方向
  Y轴正向：车身左侧
  Z轴正向：车顶向上方向

MPU6050默认方向（假设水平安装，芯片X向前、Y向左、Z向上）：
  accel_x = 纵向加速度 (+加速/-制动)
  accel_y = 横向加速度 (+左弯/-右弯)  
  accel_z = 垂向加速度 (+颠簸向上)
  gyro_x  = Roll角速度 (车身侧倾变化率)
  gyro_y  = Pitch角速度 (车身俯仰变化率)
  gyro_z  = Yaw角速度 (车身偏航变化率)
```

### D.2 互补滤波器算法

```c
// 2ms执行周期（500Hz）
// alpha = 0.98 (陀螺仪权重), dt = 0.002s

// 1. 加速度计算倾角
roll_acc  = atan2f(ay, az) * RAD2DEG;
pitch_acc = atan2f(-ax, sqrtf(ay*ay + az*az)) * RAD2DEG;

// 2. 互补滤波融合
roll  = alpha * (roll  + gx * dt) + (1-alpha) * roll_acc;
pitch = alpha * (pitch + gy * dt) + (1-alpha) * pitch_acc;

// 3. Yaw通过GPS校正（无GPS时纯积分会漂移）
yaw += gz * dt;  // 纯积分
// GPS可用时：yaw = k * yaw_gyro + (1-k) * heading_gps, k=0.95
```

### D.3 G值计算（重力补偿后）

```c
// 通过姿态角计算重力在车身坐标系的分量
gravity_x = -sinf(pitch * DEG2RAD);
gravity_y = sinf(roll * DEG2RAD) * cosf(pitch * DEG2RAD);
gravity_z = cosf(roll * DEG2RAD) * cosf(pitch * DEG2RAD);

// 重力补偿
ax_body = ax - gravity_x;
ay_body = ay - gravity_y;
az_body = az - gravity_z;

// G值计算（1g = 9.80665 m/s^2）
g_long = ax_body;      // 纵向G（+加速/-制动）
g_lat  = ay_body;      // 横向G（+左/-右）

// 10Hz低通滤波输出
g_long_lpf = g_long_lpf * 0.85 + g_long * 0.15;
g_lat_lpf  = g_lat_lpf  * 0.85 + g_lat  * 0.15;
```

### D.4 卡尔曼滤波（推荐用于赛道高精度）

状态向量：X = [速度, 加速度偏置, 位置]  
观测向量：Z = [GPS速度, GPS位置, IMU加速度]  

更新频率：预测步500Hz（IMU），校正步25Hz（GPS）。
